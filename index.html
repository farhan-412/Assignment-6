Freduently Asked Questions

1. What is the different between let, var & const?
  1. var is the old method and may cause errors.
  2. let and const are modern and safer.
  3. const is preferred when the value should not change.
2. What is the different between map(), forEach() & filter()
  1. Use map() when you want to modify each element.
  2. Use forEach() when you only need to loop through values.
  3. Use filter() when you want to extract specific elements.
3. What are arrow functions and how are they different form regular functions?
Arrow functions are a shorter way to write functions in JavaScript using the => syntax. They do not have their own this keyword and inherit it from the surrounding scope. Regular functions use the function keyword, have their own this, and can be used as constructors. Arrow functions are best for short, simple functions, while regular functions are used when more control is needed.
4. How do Javascript Promises work?
JavaScript Promises are used to handle asynchronous operations such as fetching data from a server.A Promise represents a value that may be available now, later, or never. A Promise has three states: pending (initial state), fulfilled (operation completed successfully), and rejected (operation failed). When a Promise is fulfilled, the result is handled using .then(), and when it is rejected, the error is handled using .catch(). Promises help avoid callback hell and make asynchronous code easier to read and manage.
5. How does closure work in Javascript?
A closure in JavaScript is created when a function remembers and can access variables from its outer scope, even after the outer function has finished executing. This happens because functions in JavaScript retain a reference to the environment in which they were created. Closures are commonly used for data hiding and maintaining state in programs.
